```feature
Feature: Время жизни сессии (NFR-01)
  Scenario: JWT токен имеет допустимый TTL
    Given система выдает JWT токены пользователям
    When пользователь получает JWT токен
    Then время жизни токена не должно превышать 24 часа
```

```feature
Feature: Контроль доступа (NFR-02)
  Scenario: Пользователь получает доступ к разрешенному ресурсу
    Given пользователь авторизован и имеет права доступа
    When пользователь пытается получить доступ к ресурсу
    Then доступ разрешается

  Scenario: Пользователь пытается получить доступ к запрещенному ресурсу (негативный)
    Given пользователь авторизован
    When пользователь пытается получить доступ к ресурсу без прав
    Then доступ запрещается
```
```feature
Feature: Безопасное хранение паролей (NFR-03)
  Scenario: Пароль пользователя хранится в защищенном виде
    Given пользователь регистрируется или изменяет пароль
    When пароль сохраняется в системе
    Then пароль должен быть захеширован с использованием Argon2id или bcrypt
```
```feature
Feature: Защита от SQL-инъекций (NFR-04)
  Scenario: Вредоносные SQL-запросы блокируются
    Given пользователь отправляет данные для базы
    When запрос содержит попытку SQL-инъекции
    Then запрос должен быть заблокирован и зарегистрирован
```
```feature
Feature: Логирование доступа (NFR-05)

    Given пользователь пытается получить доступ к системе

    Then событие записи в лог происходит
```

```feature
Feature: Логирование событий (NFR-06)
  Scenario: Логи сохраняются минимум 30 дней
    Given логи создаются при событиях доступа и операций
    When ведется хранение логов
    Then логи должны храниться не менее 30 дней
```
```feature
Feature: Восстановление после сбоев (NFR-07)
  Scenario: Система восстанавливается после сбоя в пределах 5 минут
    Given произошел сбой системы
    When система начинает восстановление
    Then время восстановления не должно превышать 5 минут
```

```feature
Feature: Время отклика API (NFR-08)
  Scenario: Время ответа API не превышает 300 мс на 95% запросов
    Given API доступен для клиентов
    When клиенты отправляют запросы
    Then 95% ответов должны приходить не позже 300 мс

  Scenario: Время отклика API превышает 300 мс (негативный)
    Given высокая нагрузка на API
    When клиенты отправляют запросы
    Then более 5% ответов требуют времени дольше 300 мс
```
